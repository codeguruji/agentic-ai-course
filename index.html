Foundations of AI Agents

An AI agent is more than a chatbot — it's an autonomous loop that perceives its environment, reasons about a goal, takes actions, and observes outcomes. Understanding this architecture is the foundation for everything else in this course.

What Makes an Agent?

The critical distinction between a simple LLM call and an agent is the feedback loop. A basic chat sends a prompt, gets a response, done. An agent:

  • Has access to tools — callable functions that interact with the world
  • Operates in a loop — it can take many steps to complete a task
  • Has some form of memory — context it can draw on and add to
  • Can self-correct — if a tool call fails, it reasons about why and tries again

The ReAct Pattern

ReAct (Reasoning + Acting) is the dominant paradigm for LLM-based agents. The model alternates between Thought (internal reasoning) and Action (tool invocation), observing results at each step.

text · ReAct Loop Example 
Thought: The user wants to know Apple's current stock price.
        I should search for real-time data.
Action: web_search("AAPL stock price today")
Observation: AAPL is trading at $213.50, up 1.2%

Thought: I have the price. I should also get the 52-week range
        so the user has useful context.
Action: web_search("AAPL 52 week high low 2024")
Observation: 52-week range: $164.08 – $237.23

Thought: I have everything I need. I'll compose a clear answer.
Final Answer: Apple (AAPL) is currently trading at $213.50 (+1.2%).
             Its 52-week range is $164.08 to $237.23.

Memory Types

  • In-context memory — the current conversation/messages array. Fast, but limited by the context window (200k tokens for Claude).
  • External vector memory — embeddings stored in a database like ChromaDB or Pinecone. Semantically searchable, persistent across sessions.
  • Episodic/key-value memory — structured facts stored in a database. Good for remembering specific user preferences or task outputs.

Minimal Agent Loop in Python

python · minimal_agent.py 
import anthropic
import json

client = anthropic.Anthropic()

# Define tools available to the agent
tools = [
    {
        "name": "calculator",
        "description": "Performs arithmetic operations. Use for any math.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "Math expression, e.g. '2 + 2' or '100 * 1.07'"
                }
            },
            "required": ["expression"]
        }
    }
]

def execute_tool(name: str, inputs: dict) -> str:
    if name == "calculator":
        try:
            result = eval(inputs["expression"])  # Safe in controlled env
            return str(result)
        except Exception as e:
            return f"Error: {e}"
    return "Unknown tool"

def run_agent(user_input: str):
    messages = [{"role": "user", "content": user_input}]

    while True:
        response = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=1024,
            tools=tools,
            messages=messages
        )

        # Append assistant response to history
        messages.append({"role": "assistant", "content": response.content})

        # Check if agent is done
        if response.stop_reason == "end_turn":
            for block in response.content:
                if hasattr(block, "text"):
                    print(f"Agent: {block.text}")
            break

        # Process tool calls
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                print(f"→ Calling {block.name}({block.input})")
                result = execute_tool(block.name, block.input)
                print(f"← Result: {result}")
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })

        # Feed tool results back into conversation
        messages.append({"role": "user", "content": tool_results})

# Run it
run_agent("What is (847 * 23) + (1099 / 7)? Show your work.")
💡
Pro Tip

Always add the assistant's response to your messages array before sending tool results back. Claude uses the full history to maintain coherent reasoning across multiple tool calls.

⚠️
Common Mistake

Forgetting to handle stop_reason == "tool_use" vs "end_turn". If you break the loop early, the agent never completes multi-step tasks. Always check the stop reason explicitly.

Claude Code is Anthropic's CLI-based agentic coding tool. Unlike the raw API, Claude Code operates with persistent file access, shell execution, and a built-in understanding of software projects. It turns your terminal into an autonomous engineering partner.

Claude Code uses computer use capabilities internally — it can read files, run shell commands, search the web, and edit code autonomously. It decides which tools to invoke based on your natural-language task description.

Tool Use & Function Calling

The quality of your agent is directly tied to the quality of your tool definitions. A well-designed tool schema is the difference between an agent that reliably accomplishes tasks and one that constantly makes wrong function calls.

Tool Design Principles

  • One clear responsibility per tool. Never bundle unrelated actions.
  • Descriptions are prompts. Claude reads them to decide when to call the tool. Write them like instructions, not labels.
  • Specify constraints in descriptions. "Only use when you need real-time data" prevents unnecessary calls.
  • Return structured, parseable responses. JSON is better than freeform text for tool outputs.

Robust Tool Executor

python · tools/executor.py 
import os
import json
from pathlib import Path

def execute_tool(name: str, inputs: dict) -> dict:
    """Execute a tool and return structured result."""
    try:
        if name == "read_file":
            path = Path(inputs["path"])
            if not path.exists():
                return {"error": f"File not found: {path}", "success": False}
            content = path.read_text(encoding=inputs.get("encoding", "utf-8"))
            return {
                "content": content,
                "size_bytes": path.stat().st_size,
                "success": True
            }

        elif name == "write_file":
            path = Path(inputs["path"])
            path.parent.mkdir(parents=True, exist_ok=True)
            mode = "a" if inputs.get("append") else "w"
            path.open(mode).write(inputs["content"])
            return {"path": str(path), "success": True}

        elif name == "list_directory":
            path = Path(inputs["path"])
            if inputs.get("recursive"):
                files = [str(p) for p in path.rglob("*")]
            else:
                files = [str(p) for p in path.iterdir()]
            return {"files": files, "count": len(files), "success": True}

        else:
            return {"error": f"Unknown tool: {name}", "success": False}

    except Exception as e:
        return {"error": str(e), "success": False}
💡
Pro Tip

Always return a success: bool field in tool outputs. When success is false, Claude will reason about the error and try an alternative approach instead of silently failing.

⚠️
Common Mistake

Using eval() for tool execution in production. Always validate and sanitize tool inputs. For filesystem tools, restrict paths to a designated sandbox directory to prevent path traversal attacks.

Memory & State Management

Claude has no memory between API calls. Every agent run starts fresh unless you explicitly inject context. This module covers how to give agents persistent, searchable memory across sessions.

Vector Memory with ChromaDB

bash · install 
pip install chromadb anthropic
python · memory/vector_store.py 
import chromadb
import anthropic
from datetime import datetime

client = anthropic.Anthropic()
chroma = chromadb.PersistentClient(path="./agent_memory")
collection = chroma.get_or_create_collection(
    name="agent_memories",
    metadata={"hnsw:space": "cosine"}
)

def get_embedding(text: str) -> list[float]:
    """Use Claude's embedding model via Voyage AI (recommended)
    or fall back to a simple hash-based approach for demos."""
    # Production: use voyage-3 or text-embedding-3-small
    # Demo: we'll use a placeholder (replace with real embeddings)
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=10,
        system="Return only a JSON array of 10 numbers between -1 and 1 representing a mock embedding. No explanation.",
        messages=[{"role": "user", "content": text}]
    )
    import json
    return json.loads(response.content[0].text)

def remember(text: str, metadata: dict = None):
    """Store a memory with optional metadata tags."""
    embedding = get_embedding(text)
    doc_id = f"mem_{datetime.now().timestamp()}"
    meta = {"timestamp": datetime.now().isoformat()}
    if metadata:
        meta.update(metadata)
    collection.add(
        documents=[text],
        embeddings=[embedding],
        metadatas=[meta],
        ids=[doc_id]
    )
    print(f"🧠 Stored memory: {text[:60]}...")

def recall(query: str, n_results: int = 3) -> list[str]:
    """Retrieve semantically similar memories."""
    embedding = get_embedding(query)
    results = collection.query(
        query_embeddings=[embedding],
        n_results=min(n_results, collection.count())
    )
    return results["documents"][0] if results["documents"] else []

def agent_with_memory(user_input: str):
    """Agent that injects relevant memories into each request."""
    relevant = recall(user_input)
    memory_context = ""
    if relevant:
        memory_context = "Relevant memories from past sessions:\n"
        memory_context += "\n".join(f"- {m}" for m in relevant)
        memory_context += "\n\n"

    response = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=1024,
        system=f"You are a helpful assistant with memory. {memory_context}",
        messages=[{"role": "user", "content": user_input}]
    )

    answer = response.content[0].text
    # Store this exchange as a new memory
    remember(f"User asked: {user_input} | Agent responded: {answer[:200]}")
    return answer
💡
Pro Tip

For production, use Voyage AI (Anthropic's recommended embedding partner) via voyageai package. Their voyage-3 model produces much better semantic similarity than generic embeddings for code and technical content.

Context Summarization Strategy

python · memory/summarizer.py 
MAX_CONTEXT_TOKENS = 150_000  # Leave 50k for response
SUMMARIZE_AT = 100_000

def maybe_summarize(messages: list, current_tokens: int) -> list:
    """Compress old messages if context window is filling up."""
    if current_tokens < SUMMARIZE_AT:
        return messages

    # Keep the last 10 messages verbatim; summarize everything before
    recent = messages[-10:]
    old = messages[:-10]

    summary_response = client.messages.create(
        model="claude-haiku-4-5",  # Use fast/cheap model for summarization
        max_tokens=1000,
        system="Summarize the following conversation history into a concise set of key facts and decisions. Preserve all important technical details.",
        messages=[{
            "role": "user",
            "content": f"Conversation to summarize:\n{old}"
        }]
    )

    summary_text = summary_response.content[0].text
    summary_message = {
        "role": "user",
        "content": f"[CONVERSATION SUMMARY]\n{summary_text}\n[END SUMMARY]"
    }

    print(f"📦 Compressed {len(old)} messages into summary")
    return [summary_message] + recent

Multi-Agent Systems

Complex tasks benefit from decomposition. Rather than asking one agent to do everything, you can design a network of specialized agents — each with its own system prompt, tools, and context — coordinated by an orchestrator.

Orchestrator–Worker Architecture

python · agents/multi_agent.py 
import anthropic
from dataclasses import dataclass

client = anthropic.Anthropic()

@dataclass
class AgentResult:
    agent: str
    output: str
    success: bool
    tokens_used: int

def run_specialist(
    name: str,
    system_prompt: str,
    task: str,
    tools: list = None
) -> AgentResult:
    """Run a specialist agent with a specific system prompt."""
    kwargs = {
        "model": "claude-opus-4-5",
        "max_tokens": 2048,
        "system": system_prompt,
        "messages": [{"role": "user", "content": task}]
    }
    if tools:
        kwargs["tools"] = tools

    response = client.messages.create(**kwargs)
    text = "".join(b.text for b in response.content if hasattr(b, "text"))

    return AgentResult(
        agent=name,
        output=text,
        success=True,
        tokens_used=response.usage.input_tokens + response.usage.output_tokens
    )

# ── Specialist Agents ──────────────────────────────────────────────────

def researcher(topic: str) -> str:
    result = run_specialist(
        name="Researcher",
        system_prompt="""You are a research specialist. Your job is to gather 
        comprehensive, factual information on any given topic. Structure 
        your output as: Key Facts, Statistics, Recent Developments, 
        and Sources to verify.""",
        task=f"Research the following topic thoroughly: {topic}"
    )
    print(f"✅ Researcher: {result.tokens_used} tokens")
    return result.output

def writer(research: str, format: str = "blog post") -> str:
    result = run_specialist(
        name="Writer",
        system_prompt="""You are a professional technical writer. Transform 
        research into clear, engaging content. Use concrete examples, 
        avoid jargon where possible, and maintain a confident voice.""",
        task=f"Write a {format} based on this research:\n\n{research}"
    )
    print(f"✅ Writer: {result.tokens_used} tokens")
    return result.output

def editor(draft: str) -> str:
    result = run_specialist(
        name="Editor",
        system_prompt="""You are a senior editor. Review drafts for:
        1. Factual accuracy and logical flow
        2. Clarity and concision (cut unnecessary words)
        3. Grammar and style consistency
        4. Strong opening and memorable conclusion
        Return the improved version directly with no commentary.""",
        task=f"Edit and improve this draft:\n\n{draft}"
    )
    print(f"✅ Editor: {result.tokens_used} tokens")
    return result.output

# ── Orchestrator ───────────────────────────────────────────────────────

def content_pipeline(topic: str, format: str = "blog post") -> str:
    print(f"\n🚀 Starting content pipeline for: {topic}\n")

    research = researcher(topic)
    draft = writer(research, format)
    final = editor(draft)

    print("\n✨ Pipeline complete!")
    return final

# Run it
output = content_pipeline(
    topic="The impact of MCP (Model Context Protocol) on AI agent development",
    format="technical blog post"
)
print(output)
💡
Pro Tip

Use claude-haiku-4-5 for fast, cheap specialist agents that do simple transformations (summarize, format, validate). Reserve claude-opus-4-5 for the orchestrator and complex reasoning tasks. This can reduce costs by 10–20x.

Parallel Agent Execution

python · parallel agents 
import asyncio
import anthropic

async_client = anthropic.AsyncAnthropic()

async def run_parallel_research(topics: list[str]) -> list[str]:
    """Research multiple topics concurrently."""
    async def research_one(topic: str) -> str:
        response = await async_client.messages.create(
            model="claude-haiku-4-5",
            max_tokens=500,
            messages=[{"role": "user", "content": f"Summarize key facts about: {topic}"}]
        )
        return response.content[0].text

    results = await asyncio.gather(*[research_one(t) for t in topics])
    return list(results)

# Research 5 topics in parallel instead of sequentially

results = asyncio.run(run_parallel_research(topics))

MCP — Model Context Protocol

MCP is an open protocol that standardizes how AI models connect to external data sources and tools. Think of it as USB-C for AI integrations — one standard connector that works across all compatible apps and services.

Core Concepts

  • MCP Server — a process that exposes tools, resources, and prompts via the MCP protocol
  • MCP Client — a host app (like Claude Code) that connects to MCP servers
  • Tools — callable functions the LLM can invoke via the server
  • Resources — data sources the LLM can read (files, databases, APIs)

import asyncio
import httpx
from mcp.server import Server
from mcp.server.stdio import stdio_server
import mcp.types as types

server = Server("weather-tools")

@server.list_tools()
async def handle_list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="get_current_weather",
            description="Get current weather conditions for any city. Returns temperature, conditions, humidity, and wind speed.",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "City name, e.g. 'London' or 'Tokyo'"},
                    "units": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "default": "celsius"
                    }
                },
                "required": ["city"]
            }
        ),
        types.Tool(
            name="get_forecast",
            description="Get 5-day weather forecast for a city.",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {"type": "string"},
                    "days": {"type": "integer", "default": 5, "minimum": 1, "maximum": 7}

Retry with Exponential Backoff

python · production/resilient_agent.py 
import anthropic
import logging
import time
from tenacity import (
    retry, stop_after_attempt, wait_exponential,
    retry_if_exception_type, before_sleep_log
)

logger = logging.getLogger(__name__)
client = anthropic.Anthropic()

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=4, max=60),
    retry=retry_if_exception_type((
        anthropic.RateLimitError,
        anthropic.APIConnectionError,
        anthropic.InternalServerError
    )),
    before_sleep=before_sleep_log(logger, logging.WARNING)
)
def resilient_call(messages: list, tools: list = None, model: str = "claude-opus-4-5"):
    kwargs = {
        "model": model,
        "max_tokens": 4096,
        "messages": messages
    }
    if tools:
        kwargs["tools"] = tools

    response = client.messages.create(**kwargs)
    logger.info(f"API call successful. Tokens: in={response.usage.input_tokens} out={response.usage.output_tokens}")
    return response

import json
import time
import uuid
from contextlib import contextmanager
from dataclasses import dataclass, asdict, field
from datetime import datetime

@dataclass
class AgentTrace:
    trace_id: str = field(default_factory=lambda: str(uuid.uuid4())[:8])
    task: str = ""
    model: str = ""
    started_at: str = ""
    ended_at: str = ""
    duration_ms: int = 0
    total_tokens: int = 0
    estimated_cost_usd: float = 0.0
    tool_calls: list = field(default_factory=list)
    steps: int = 0
    success: bool = False
    error: str = ""

# Token costs per million (Claude pricing, verify current rates)
COST_PER_M = {
    "claude-opus-4-5": {"input": 15.0, "output": 75.0},
    "claude-haiku-4-5": {"input": 0.80, "output": 4.0},
}

@contextmanager
def trace_agent(task: str, model: str = "claude-opus-4-5"):
    trace = AgentTrace(
        task=task,
        model=model,
        started_at=datetime.now().isoformat()
    )
    start = time.time()

    try:
        yield trace
        trace.success = True
    except Exception as e:
        trace.error = str(e)
        raise
    finally:
        trace.ended_at = datetime.now().isoformat()
        trace.duration_ms = int((time.time() - start) * 1000)
        # Emit structured log
        print(json.dumps(asdict(trace), indent=2))

    """Stream agent responses using Server-Sent Events."""
    async def event_generator():
        with client.messages.stream(
            model=request.model,
            max_tokens=4096,
            messages=[{"role": "user", "content": request.task}]

from agents.researcher import ResearchAgent
from agents.analyst import AnalystAgent
from agents.writer import WriterAgent
from memory.vector_store import remember, recall
from production.observability import trace_agent

def run_research_pipeline(topic: str) -> dict:
    with trace_agent(task=topic) as trace:

        # 1. Check memory for prior research on this topic
        prior_research = recall(topic, n_results=3)

        # 2. Researcher gathers new information
        researcher = ResearchAgent()
        raw_findings = researcher.gather(topic, context=prior_research)
        trace.steps += 1

        # 3. Analyst extracts structured facts and insights
        analyst = AnalystAgent()
        analysis = analyst.analyze(raw_findings)
        trace.steps += 1

        # 4. Writer composes the final report
        writer = WriterAgent()
        report = writer.compose(analysis, topic)
        trace.steps += 1

        # 5. Store in memory for future queries
        remember(
            f"Research on {topic}: {report[:500]}",
            metadata={"topic": topic, "type": "research_report"}
        )

        return {
            "topic": topic,
            "report": report,
            "trace_id": trace.trace_id,
            "steps": trace.steps
        }
